For the purpose of this exercise, let's assume that application language in this context is Java.

For linting in this application, we could consider using tools like SonarQube, SpotBugs and CheckStyle.
For testing our Java application, we would consider using JUnit which is the most popular or Mockito and TestNG as alternatives.
When it comes to building the project in Java, tools like Maven or Gradle are usually used. Bazel could also be considered as an option.

Alternatives to CI setups besides Jenkins and GitHub Actions include GitLab CI/CD, Azure Devops and Amazon AWS.

For this project, we would prefer to host it in a cloud-based environment. This is becasue the team is small and the project is most likely not a large one so going through the hassle of managing the infrastructure by ourself would be unnecessary. Additionally, a cloud solution wold be much easier to maintain and scale up as we make progres in our application. It would apparently be much cheaper for us to adopt the idea of hosting our application in a cloud-based environment.
